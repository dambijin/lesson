단계별로 풀어보자
---
회원가입 front
회원가입 양식 : 아이디, 비밀번호, 비밀번호 확인, 가입하기 버튼
검증
+ 가입하기 버튼 클릭 시 모두 검증
+ 각 input의 blur 이벤트 시점에서 개별 검증
+ 각 input의 아래 또는 옆에 검증 실패 내용 표시
검증 내용
+ 아이디 : 비어있지 않는지 확인
+ 비밀번호 확인 : 비밀번호와 정확히 일치하는지 확인
검증 통과 시 학습도에 따라 선택
alert으로 아이디, 비밀번호 표시
서블릿/컨트롤러로 전송 후 출력
java에서 동일한 검증
DB에 저장
---
로그인
아이디, 비밀번호를 변수에 저장하고 출력 만(판단하지 않음)
학습도에 따라 다음 단계 선택 후 "성공", "실패" 출력
지역변수, 필드 등에 "admin", "1234"와 같이 미리 저장해둔 값과 전달받은 값이 모두 같은지 판단
DB에서 아이디, 비밀번호가 있는지 확인
---
세션을 이용한 로그인 유지
로그인한 사용자만 회원 전용 페이지에 접근할 수 있도록 구현
구현 순서 예시
회원 전용 페이지를 표시하는 서블릿/컨트롤러 구현
로그인을 성공한 경우에만 세션에 저장
위 1.에서 세션에 해당 내용이 있는 경우만 페이지 표시하도록 수정
filter를 사용해서 로그인 체크
---
모든 사람이 목록을 CRUD(Create, Read, Update, Delete)할 수 있는 페이지
로그인하지 않아도 모두가 모든 목록을 CRUD 할 수 있게 구현
로그인 한 사람만 전체 목록을 CUD, 로그인 하지 않아도 Read 가능하게.

---
장바구니 또는 좋아요

장바구니 또는 좋아요를 확인 할 수 있는 페이지 제공
---
페이징 구현
한 페이지에 10개씩(혹은 선택할 수 있게)
구성 : (이전) (1) (2) (3) (4) (5) (다음)
한번에 보여주는 하단의 페이지의 개수는 적당하게(3, 5, 10개씩)
이전/다음은 하단 페이지 덩어리의 이동
+ 5개씩 보여주는 경우 : 다음 클릭 시 6 7 8 9 10 표시
